## Идея

Интерактивная веб-карта мирового океана с двумя режимами:

1. **Визуализация** реальных течений мирового океана (исторические данные + квази-реальное время)
2. **Симуляция** — пользователь кликает на карту, выбирает тип катастрофы (нефть, пластик, химия, радиация) и наблюдает как вещество распространяется по течениям

**Почему интересно для портфолио:**

- Визуально впечатляет, понятно без технического бэкграунда
- Реальные научные данные и верифицируемые модели
- Охватывает весь стек data engineer: ingestion → storage → processing → serving → real-time
- Уникально — аналоги (NOAA WebTAP, GNOME) существуют, но уродливые, для спасателей, не для людей
- Можно воспроизвести реальные катастрофы (Deepwater Horizon 2010, Exxon Valdez, Фукусима) и сравнить симуляцию с реальностью

## Источники данных

|Источник|Что даёт|Доступ|
|---|---|---|
|**CMEMS (Copernicus)**|Течения U/V, температура, солёность. Глобально, 1/12°, с 1993 года|Бесплатно, Python SDK `copernicusmarine`, нужна регистрация|
|**NOAA OSCAR**|Поверхностные течения каждые 5 дней|Бесплатно, HTTP, без регистрации|
|**HYCOM**|Глобальная модель океана от ВМС США|Бесплатно, HTTP|

**Формат данных:** NetCDF — 4D массив (время, глубина, lat, lon). Работать через Xarray + Dask.

**Важная оговорка:** earth.nullschool.net не источник данных — это визуализатор поверх тех же CMEMS/NOAA. Их закрытый формат `.epak` недоступен публично.

## Архитектура

### Часть 1 — Data Pipeline (течения)

```
CMEMS / OSCAR / HYCOM
        │
   Airflow DAG (ежедневно)
        │
   S3 / MinIO
   ├── raw/       ← оригинальные NetCDF
   ├── processed/ ← Iceberg таблицы
   │   ├── currents/ (date, lat, lon, u_velocity, v_velocity, depth)
   │   └── sst/      (date, lat, lon, temperature)
   └── tiles/     ← предпросчитанные векторные тайлы для фронтенда
        │
   Xarray + Dask (обработка NetCDF)
        │
   dbt (трансформации, агрегаты, сезонность)
        │
   PostGIS + FastAPI (REST + Vector Tiles)
```

### Часть 2 — Simulation Engine (разлив)

```
Клик пользователя на карту
        │
   POST /api/simulate
   (тип вещества, объём, дата)
        │
   Celery Worker
        │
   OpenDrift (Lagrangian particle tracking)
   + данные течений из Iceberg
   + диффузия + испарение + ветер
        │
   Redis (кадры симуляции)
        │
   FastAPI WebSocket
        │
   Deck.gl (ScatterplotLayer — частицы в реальном времени)
```

### Полный стек

|Слой|Технологии|
|---|---|
|Ingestion|Airflow + Python + `copernicusmarine` SDK|
|Raw Storage|S3 / MinIO (NetCDF)|
|Table Format|Apache Iceberg (time travel, partition pruning по дате/региону)|
|Processing|Xarray + Dask (NetCDF), dbt (трансформации)|
|Simulation|OpenDrift (open source, используется береговой охраной Норвегии)|
|Queue|Redis + Celery|
|Serving|FastAPI (REST + WebSocket)|
|Visualization|Deck.gl (WebGL) — VelocityLayer для течений, ScatterplotLayer для частиц|
|Frontend|React|
|Orchestration|Airflow|
|IaC|Terraform + Docker|
|Quality|Great Expectations|

## Конкурентный анализ

|Аналог|Проблема|
|---|---|
|NOAA WebTAP|Только нефть, только США, интерфейс 2005 года|
|GNOME Suite|Десктоп, для профессионалов-спасателей|
|earth.nullschool.net|Только визуализация, нет интерактивной симуляции|
|MyOcean (CMEMS)|Научный инструмент, не для широкой аудитории|

**Незанятая ниша:** красивый, понятный, глобальный симулятор с несколькими типами веществ и воспроизведением реальных катастроф.

---

## Уникальные фичи для выделения проекта

1. **Несколько типов веществ** — нефть, пластик (микропластик), химия, радиация
2. **Исторический режим** — воспроизвести Deepwater Horizon 2010, Exxon Valdez 1989, слив воды с Фукусимы 2023
3. **Верификация** — сравнить симуляцию с реальными данными о загрязнении (делает проект научно честным)
4. **Data engineering акцент** — в отличие от аналогов, архитектура pipeline полностью прозрачна и задокументирована
