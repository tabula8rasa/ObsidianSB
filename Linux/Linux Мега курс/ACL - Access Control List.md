ACL — это **Access Control List**, то есть **список контроля доступа**.

В Linux ACL нужен для случаев, когда обычных прав `rwx` уже не хватает.

## Сначала проблема обычных прав

У файла в Linux обычно есть только 3 набора прав:

- для **владельца**
    
- для **группы**
    
- для **всех остальных**
    

Например:

```bash
-rw-r-----
```

Это означает:

- владелец может читать и писать
    
- группа может читать
    
- остальные ничего не могут
    

Но тут нельзя гибко сказать:

- пользователю `ivan` дать `rw`
    
- пользователю `oleg` дать только `r`
    
- группе `dev` дать `rw`
    

Обычная модель `owner / group / others` слишком грубая.

---

## Что делает ACL

ACL позволяет назначать права **конкретным пользователям и группам** для конкретного файла или директории.

То есть можно сказать:

- этому пользователю — чтение и запись
    
- этой группе — только чтение
    
- остальным — как раньше
    

И всё это на одном объекте.

---

## Простой пример

Есть файл `report.txt`.

Обычными правами ты можешь задать что-то вроде:

```bash
-rw-r-----
```

Но ты хочешь, чтобы:

- владелец имел `rw`
    
- группа имела `r`
    
- пользователь `ivan` тоже имел `rw`, хотя он не владелец и не в нужной группе
    

Вот тут и нужен ACL.

---

## Как посмотреть ACL

```bash
getfacl report.txt
```

Пример вывода:

```text
user::rw-
user:ivan:rw-
group::r--
mask::rw-
other::---
```

Разбор:

- `user::rw-` — права владельца
    
- `user:ivan:rw-` — отдельные права для пользователя `ivan`
    
- `group::r--` — права группы файла
    
- `mask::rw-` — максимальные права для именованных пользователей и групп ACL
    
- `other::---` — права для остальных
    

---

## Как выдать права через ACL

Например, дать пользователю `ivan` чтение и запись:

```bash
setfacl -m u:ivan:rw report.txt
```

Дать группе `dev` право только читать:

```bash
setfacl -m g:dev:r report.txt
```

Посмотреть результат:

```bash
getfacl report.txt
```

---

## ACL у директории

ACL можно ставить и на директорию.

Например:

```bash
setfacl -m u:ivan:rwx project_dir
```

Это даст пользователю `ivan` доступ к папке.

Но тут важно помнить:

- для директории `r` — смотреть список файлов
    
- `w` — создавать и удалять файлы
    
- `x` — входить в директорию и обращаться к объектам внутри
    

Без `x` доступ к каталогу обычно бесполезен.

---

## Default ACL

Можно задать **ACL по умолчанию** для директории. Тогда новые файлы и папки внутри будут автоматически наследовать эти правила.

Пример:

```bash
setfacl -d -m u:ivan:rwx project_dir
```

Это значит:

- всё новое внутри `project_dir` будет создаваться с ACL для `ivan`
    

Это очень удобно для общих рабочих папок.

---

## Чем ACL отличается от chmod

`chmod` работает только с:

- владельцем
    
- группой
    
- остальными
    

ACL позволяет делать тонкую настройку:

- отдельным пользователям
    
- отдельным группам
    
- сразу нескольким субъектам
    

То есть:

- `chmod` — базовые права
    
- `ACL` — расширенные права
    

---

## Как понять, что у файла есть ACL

Если сделать:

```bash
ls -l report.txt
```

и увидеть, например:

```bash
-rw-r-----+ 1 user user ...
```

знак **`+`** в конце прав означает, что у файла есть ACL.

---

## Важный момент про mask

В ACL есть поле `mask`.

Оно ограничивает максимальные права для:

- именованных пользователей
    
- именованных групп
    
- группы файла
    

Например, ты можешь выдать `ivan:rw`, но если `mask` стоит `r--`, то эффективно у него будет только чтение.

Это одна из самых частых причин путаницы в ACL.

---

## Когда ACL реально нужен

ACL полезен, когда:

- несколько людей работают с одним файлом или папкой
    
- не хочется менять владельца или основную группу
    
- надо точечно выдать доступ одному пользователю
    
- нужно настроить общую папку проекта с наследованием прав
    

---

## Самая короткая формулировка

**ACL — это расширенная система прав доступа в Linux, которая позволяет выдавать права не только владельцу, группе и остальным, но и конкретным пользователям и группам.**

## Мини-пример в одной строке

```bash
setfacl -m u:ivan:rw file.txt
```

Это значит:  
**дать пользователю `ivan` права чтения и записи на `file.txt`.**

Могу еще сразу показать разницу между **chmod 755**, **umask** и **ACL** на одном простом примере.
