## 1. Введение в Bash-скрипты

**Bash-скрипт** — это набор команд оболочки, объединённых общей целью для многократного использования.

Скрипты удобны, когда нужно:

- автоматизировать рутинные действия;
    
- запускать несколько команд подряд;
    
- обрабатывать файлы по шаблону;
    
- делать резервные копии, проверки, развёртывание;
    
- писать простые утилиты без полноценного языка программирования.
    

### Виды сценариев

- **Исполняемые файлы** — скрипты, сохранённые в отдельный файл для частого запуска.
    
- **Однострочники** — одноразовые сценарии, запускаемые прямо в командной строке.
    

Пример однострочника:

```bash
for f in *.log; do gzip "$f"; done
```

### Создание и запуск

Файл скрипта создаётся в любом текстовом редакторе, например `vim`, `nano`, `nvim`.

Для запуска есть два основных варианта:

1. Через интерпретатор:
    

```bash
bash script.sh
```

2. Как исполняемый файл:
    

```bash
chmod +x script.sh
./script.sh
```

### Важное различие

- `bash script.sh` запускает файл через `bash`, даже если у файла нет права на исполнение.
    
- `./script.sh` требует права `x` и обычно использует shebang.

## 2. Структура скрипта и Shebang

Каждый скрипт обычно начинается со специальной строки — **shebang**.

### Синтаксис

```bash
#!/bin/bash
```

Она сообщает системе, какой интерпретатор использовать для выполнения файла.

### Портативность

Часто рекомендуют писать:

```bash
#!/usr/bin/env bash
```

Такой вариант более переносим, потому что `env` ищет `bash` через переменную `PATH`. Это особенно полезно, если интерпретатор находится не по стандартному пути.

### Пример полного минимального скрипта

```bash
#!/usr/bin/env bash
echo "Hello, world!"
```

### Комментарии

Комментарии начинаются с `#`:

```bash
# Это комментарий
echo "test"
```

## 3. Переменные и типы данных

В Bash нет строгой типизации в привычном смысле: почти всё воспринимается как **строка**.

### Задание переменной

Пробелов вокруг `=` быть не должно:

```bash
NAME=Ivan
```

Неправильно:

```bash
NAME = Ivan
```

### Обращение к переменной

Используется `$NAME` или `${NAME}`:

```bash
echo $NAME
echo ${NAME}
```

Форма `${NAME}` удобнее, когда рядом стоит другой текст:

```bash
FILE="report"
echo "${FILE}.txt"
```

### Вывод команды в переменную

Современный рекомендуемый синтаксис:

```bash
DATE=$(date)
echo "$DATE"
```

Старый вариант с обратными кавычками тоже работает:

```bash
DATE=`date`
```

Но он хуже читается и плохо вкладывается друг в друга.

### Экранирование

Чтобы вывести символ `$` буквально:

```bash
echo "\$HOME"
```

### Важное замечание

Если значение содержит пробелы, переменную почти всегда надо брать в двойные кавычки:

```bash
FILE="my document.txt"
cat "$FILE"
```

Без кавычек Bash разобьёт строку на несколько аргументов.

## 4. Кавычки в Bash

### Двойные кавычки `" "`

Позволяют раскрывать переменные и подстановки команд:

```bash
NAME="Alex"
echo "Hello, $NAME"
```

### Одинарные кавычки `' '`

Воспринимают содержимое буквально:

```bash
NAME="Alex"
echo 'Hello, $NAME'
```

Выведется именно `$NAME`, а не значение переменной.

### Обратные кавычки `` ` ` ``

Используются для подстановки команды:

```bash
echo `date`
```

Но лучше писать так:

```bash
echo "$(date)"
```

### Практический вывод

- для обычного текста без раскрытия — одинарные кавычки;
    
- для строк с переменными — двойные;
    
- для команд — `$(...)`.

## 5. Арифметические операции

Bash умеет работать с целыми числами.

### Способ 1. `let`

```bash
let "A = 2 + 2"
echo "$A"
```

### Способ 2. Двойные круглые скобки

```bash
(( B = 2 + 3 ))
echo "$B"
```

Внутри `(( ))` можно использовать переменные без `$`:

```bash
X=10
Y=5
(( Z = X + Y ))
echo "$Z"
```

### Инкремент и декремент

```bash
COUNT=0
(( COUNT++ ))
(( COUNT+=5 ))
echo "$COUNT"
```

### Важное ограничение

Обычная арифметика Bash — **целочисленная**. Для дробных вычислений часто используют `bc`:

```bash
echo "scale=2; 5/2" | bc
```

## 6. Аргументы и переменные окружения

### Аргументы командной строки

При запуске скрипта:

```bash
./script.sh one two three
```

внутри:

- `$1` = `one`
    
- `$2` = `two`
    
- `$3` = `three`
    

Пример:

```bash
echo "Первый аргумент: $1"
echo "Второй аргумент: $2"
```

### Специальные переменные

- `$0` — имя скрипта;
    
- `$#` — количество аргументов;
    
- `$?` — код возврата предыдущей команды;
    
- `$$` — PID текущего shell-процесса;
    
- `$@` — все аргументы по отдельности;
    
- `$*` — все аргументы одной строкой.
    

Пример:

```bash
echo "Имя скрипта: $0"
echo "Количество аргументов: $#"
echo "PID: $$"
```

### Аргументы больше 9

Нужно использовать фигурные скобки:

```bash
echo "${10}"
```

### Переменные окружения

Это глобальные переменные, которые уже доступны процессу:

```bash
echo "$HOME"
echo "$PATH"
echo "$USER"
```

Можно передать переменную только на один запуск:

```bash
MODE=debug ./script.sh
```

А внутри скрипта прочитать:

```bash
echo "$MODE"
```

## 7. Условия и команда `test`

В Bash код возврата `0` означает **успех**, а не ноль в математическом смысле.  
Любое ненулевое значение означает ошибку или ложь.

### `test` и `[ ]`

Примеры:

```bash
test -f file.txt
[ -f file.txt ]
```

### Частые проверки

- `-z "$VAR"` — строка пустая;
    
- `-n "$VAR"` — строка не пустая;
    
- `-e file` — объект существует;
    
- `-f file` — обычный файл;
    
- `-d dir` — директория;
    
- `-r file` — доступен для чтения;
    
- `-w file` — доступен для записи;
    
- `-x file` — исполняемый.
    

Для чисел:

- `-eq` — равно;
    
- `-ne` — не равно;
    
- `-gt` — больше;
    
- `-lt` — меньше;
    
- `-ge` — больше или равно;
    
- `-le` — меньше или равно.
    

### `[[ ]]`

Это более удобная bash-конструкция:

```bash
[[ -f file.txt && -r file.txt ]]
```

Она безопаснее и удобнее, чем старый вариант `[ ]`, особенно при работе со строками и шаблонами.

### Конструкция `if`

```bash
if [ -f /etc/passwd ]; then
    echo "Файл существует"
else
    echo "Файл не найден"
fi
```

### Пример со строкой

```bash
NAME="Ivan"
if [[ "$NAME" == "Ivan" ]]; then
    echo "Привет, Ivan"
fi
```

### Проверка кода возврата

```bash
cp file1.txt file2.txt
if [ $? -eq 0 ]; then
    echo "Копирование успешно"
else
    echo "Ошибка копирования"
fi
```

Но чаще пишут так:

```bash
if cp file1.txt file2.txt; then
    echo "Копирование успешно"
else
    echo "Ошибка копирования"
fi
```

## 8. Циклы

### `for`

Цикл по списку значений:

```bash
for x in one two three; do
    echo "$x"
done
```

Цикл по файлам:

```bash
for f in *.txt; do
    echo "Файл: $f"
done
```

Цикл по диапазону:

```bash
for i in {1..5}; do
    echo "$i"
done
```

### `while`

Выполняется, пока условие истинно:

```bash
COUNT=1
while [ $COUNT -le 5 ]; do
    echo "$COUNT"
    (( COUNT++ ))
done
```

### `until`

Выполняется, пока условие ложно:

```bash
COUNT=1
until [ $COUNT -gt 5 ]; do
    echo "$COUNT"
    (( COUNT++ ))
done
```

### `IFS`

`IFS` — разделитель полей. По умолчанию это пробел, таб и перевод строки.

Для построчного чтения часто используют:

```bash
while IFS= read -r line; do
    echo "$line"
done < file.txt
```

Это безопаснее, чем `for line in $(cat file.txt)`, потому что там строки ломаются по пробелам.

## 9. Функции

Функции нужны для повторного использования кода.

### Объявление

```bash
say_hello() {
    echo "Hello, $1"
}
```

Вызов:

```bash
say_hello Ivan
```

### Аргументы функции

Они доступны так же, как и в скрипте:

- `$1`
    
- `$2`
    
- `$#`
    

Пример:

```bash
sum_two() {
    (( result = $1 + $2 ))
    echo "$result"
}
sum_two 3 5
```

### `return`

`return` возвращает **код завершения**, а не обычное строковое значение:

```bash
check_file() {
    [ -f "$1" ]
    return $?
}
```

Если нужно “вернуть текст”, обычно используют `echo`:

```bash
make_name() {
    echo "user_$1"
}
NAME=$(make_name 10)
echo "$NAME"
```

### Локальные переменные

Внутри функции полезно использовать `local`:

```bash
myfunc() {
    local name="$1"
    echo "Hello, $name"
}
```

## 10. Отладка (Debug)

Для отладки Bash-скриптов часто используют ключ `-x`:

```bash
bash -x script.sh
```

Тогда в `stderr` выводятся все исполняемые команды после раскрытия переменных.

### Включить отладку внутри скрипта

```bash
set -x
```

Отключить:

```bash
set +x
```

### Ещё полезные режимы

- `set -e` — завершать скрипт при ошибке команды;
    
- `set -u` — считать ошибкой использование необъявленной переменной;
    
- `set -o pipefail` — считать конвейер ошибочным, если упала любая команда внутри.
    

Часто в начале серьёзного скрипта пишут:

```bash
set -euo pipefail
```

Это делает скрипт надёжнее, но с таким режимом нужно аккуратно писать условия и проверки.

## 11. Практические советы по написанию Bash-скриптов

### Всегда брать переменные в кавычки

Правильно:

```bash
rm -- "$file"
```

Неправильно:

```bash
rm $file
```

Если в имени файла есть пробел, второй вариант может сломаться.

### Использовать `$(...)`, а не обратные кавычки

Так код читается лучше:

```bash
FILES=$(ls)
```

### Проверять входные аргументы

Пример:

```bash
if [ $# -lt 1 ]; then
    echo "Использование: $0 имя_файла"
    exit 1
fi
```

### Использовать `exit`

Чтобы завершить скрипт с кодом:

```bash
exit 0
exit 1
```

### Полезный шаблон простого скрипта

```bash
#!/usr/bin/env bash
set -euo pipefail

if [ $# -lt 1 ]; then
    echo "Использование: $0 имя"
    exit 1
fi

name="$1"
echo "Привет, $name"
```

## 12. Короткие выводы

1. Bash-скрипт — это набор команд оболочки для автоматизации задач.
    
2. Скрипт обычно начинается с shebang.
    
3. В Bash почти всё работает как строки, но есть встроенная целочисленная арифметика.
    
4. Аргументы скрипта передаются через `$1`, `$2`, а служебные данные доступны через `$0`, `$#`, `$?`, `$$`.
    
5. Условия строятся через `test`, `[ ]` или `[[ ]]`.
    
6. Основные циклы — `for`, `while`, `until`.
    
7. Код удобно выносить в функции.
    
8. Для отладки очень полезны `bash -x` и `set -x`.
    
9. Для более надёжных скриптов часто используют `set -euo pipefail`.
    
10. Перед написанием сложного скрипта стоит проверить, нет ли уже готовой утилиты, которая решает задачу проще и надёжнее.
