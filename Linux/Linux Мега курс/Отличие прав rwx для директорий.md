**Директория — это таблица вида “имя файла -> inode”.**  
Поэтому права на директорию управляют не “содержимым файлов”, а **входом в каталог, просмотром списка и изменением списка объектов**.

---

## Что значит `r` для директории

`r` = **read** = можно **прочитать список имён** в каталоге.

То есть можно сделать:

```bash
ls dir
```

и увидеть, что там лежит:

- `file1`
    
- `notes.txt`
    
- `subdir`
    

Но одного `r` недостаточно для полноценной работы.

---

## Что значит `x` для директории

`x` для директории — это не “запуск”, а **search / traverse**, то есть:

- можно **зайти** в директорию через `cd`
    
- можно обращаться к объектам внутри **по имени**, если имя известно
    
- можно проходить через каталог как через часть пути
    

Примеры:

```bash
cd dir
cat dir/file.txt
ls -l dir/file.txt
```

Без `x` ты не сможешь нормально использовать каталог как часть пути, даже если знаешь имя файла.

---

## Что значит `w` для директории

`w` = можно **менять содержимое каталога как списка записей**:

- создавать файлы
    
- удалять файлы
    
- переименовывать файлы
    
- создавать подкаталоги
    
- удалять подкаталоги
    

Но почти всегда `w` имеет смысл только **вместе с `x`**.

---

# Самые важные комбинации

## `r--` на директории

Можно прочитать список имён, но нельзя зайти в каталог и нормально работать с объектами внутри.

То есть увидеть список иногда можно, а открыть файлы — нет.

---

## `--x` на директории

Очень интересный случай.

Можно:

- зайти в каталог
    
- обратиться к файлу, **если знаешь его имя**
    

Но нельзя:

- посмотреть список файлов через обычный `ls dir`
    

Пример идеи:

- `cat dir/known_file.txt` может сработать
    
- `ls dir` — нет
    

То есть каталог “проходимый, но не обозримый”.

---

## `r-x` на директории

Это самый обычный режим для чтения каталога.

Можно:

- зайти в каталог
    
- посмотреть список файлов
    
- читать файлы внутри, если права на сами файлы позволяют
    

Но нельзя:

- создавать
    
- удалять
    
- переименовывать записи в каталоге
    

---

## `-wx` на директории

Можно менять записи каталога, но нельзя читать список имён.

То есть ситуация странная, но возможная:

- можно удалить или создать файл по известному имени,
    
- но нельзя удобно посмотреть, что там вообще лежит.
    

---

## `rwx` на директории

Полный доступ:

- смотреть список,
    
- заходить,
    
- создавать,
    
- удалять,
    
- переименовывать.
    

---

# Очень важное отличие от файлов

## Удаление файла зависит в первую очередь от прав на директорию, а не на сам файл

Это одна из самых частых ловушек.

Например, пусть файл:

```bash
-r--r--r--
```

То есть сам файл вообще не writable.

Но если у тебя есть права `wx` на директорию, где он лежит, ты всё равно можешь **удалить** этот файл.

Почему?  
Потому что удаление — это не “изменение файла”, а **удаление записи о нём из каталога**.

То же самое с переименованием.

# Примеры по операциям

Пусть есть каталог `docs`.

## 1. Просмотр списка файлов

Нужны:

- `r` на директорию
    
- обычно ещё `x` на директорию
    

```bash
ls docs
```

---

## 2. Войти в каталог

Нужно:

- `x`
    

```bash
cd docs
```

---

## 3. Открыть файл по известному пути

Нужно:

- `x` на все каталоги в пути
    
- права на сам файл отдельно
    

Например:

```bash
cat docs/report.txt
```

Чтобы это сработало:

- нужен `x` на `docs`
    
- нужен `r` на `report.txt`
    

---

## 4. Создать новый файл

Нужны:

- `w` и `x` на директорию
    

```bash
touch docs/new.txt
```

---

## 5. Удалить файл

Нужны:

- `w` и `x` на директорию
    

```bash
rm docs/old.txt
```

Права самого `old.txt` тут не главные.

---

## 6. Переименовать файл

Нужны:

- `w` и `x` на директорию
    

```bash
mv docs/a.txt docs/b.txt
```

---

# Почему для директории нужен `x`, а не только `r`

Потому что:

- `r` — это “я вижу список имён”
    
- `x` — это “я могу пользоваться каталогом как частью пути”
    

Без `x` каталог как будто виден снаружи, но пройти через него нельзя.

---

# Простой образ

Представь, что директория — это шкаф с ящиками:

- `r` — можно прочитать список наклеек на ящиках
    
- `x` — можно открыть шкаф и дотянуться до нужного ящика
    
- `w` — можно добавлять, убирать и переименовывать ящики
    

---

# Ещё один важный момент — sticky bit

Для общих директорий, например `/tmp`, часто ставят **sticky bit**.

Тогда даже если у всех есть `rwx`, удалять чужие файлы нельзя.  
Удалить может только:

- владелец файла
    
- владелец директории
    
- root
---

# Короткая таблица

|Право|Для файла|Для директории|
|---|---|---|
|`r`|читать содержимое файла|видеть список имён в каталоге|
|`w`|менять содержимое файла|создавать, удалять, переименовывать записи|
|`x`|запускать файл|входить в каталог и проходить через него|
