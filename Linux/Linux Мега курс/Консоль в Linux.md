## 1. Преимущества командной строки и оболочка Bash

Использование терминала часто эффективнее графического интерфейса, так как позволяет работать исключительно с клавиатуры, не отвлекаясь на мышь. В большинстве дистрибутивов в качестве основного интерпретатора команд используется **Bash**.

Командная строка особенно удобна, когда нужно:

- быстро выполнять однотипные действия;
    
- обрабатывать много файлов;
    
- автоматизировать работу через скрипты;
    
- управлять удалённой машиной по SSH;
    
- комбинировать простые команды в сложные цепочки.
    

#### Конфигурационные файлы Bash:

- **`/etc/profile`**: системный файл, общий для всех пользователей. В нём настраиваются глобальные переменные окружения, например `PATH`, и параметры вроде `umask`.
    
- **`~/.bash_profile`**: пользовательский файл в домашней директории; выполняется при первом логине в систему. Здесь задаются индивидуальные переменные окружения и команды, которые должны запускаться один раз при входе.
    
- **`~/.bashrc`**: выполняется при запуске каждого нового экземпляра Bash. В нём обычно прописываются **алиасы**, настройки приглашения shell, автодополнение и пользовательские функции.
    

### Дополнительно

Часто встречается такая схема:

- `~/.bash_profile` загружает `~/.bashrc`;
    
- все интерактивные настройки держат именно в `~/.bashrc`.
    

Пример:

```bash
if [ -f ~/.bashrc ]; then
    . ~/.bashrc
fi
```

### Полезные команды

Проверить, какой shell используется:

```bash
echo $SHELL
```

Посмотреть значение переменной `PATH`:

```bash
echo $PATH
```

Добавить алиас:

```bash
alias ll='ls -lah'
```

Сделать алиас постоянным:

```bash
echo "alias ll='ls -lah'" >> ~/.bashrc
source ~/.bashrc
```

## 2. Горячие клавиши (Hotkeys)

Для ускорения работы в Bash рекомендуется использовать следующие сочетания:

- **`Ctrl + стрелки`**: перемещение курсора по словам.
    
- **`Ctrl + W`**: удаление слова слева от курсора.
    
- **`Ctrl + R`**: поиск по истории команд.
    
- **`Alt + .`**: подстановка последнего аргумента из предыдущей команды.
    
- **`Ctrl + X + E`**: открытие текущей команды в текстовом редакторе по умолчанию для удобного редактирования длинных строк.
    
- **`Ctrl + Z`**: приостановка текущего процесса.
    

### Важное уточнение

Обычно при нажатии `Ctrl + Z` терминал отправляет процессу **`SIGTSTP`**, а не `SIGSTOP`. На практике это воспринимается как “поставить процесс на паузу”. После этого можно:

- вернуть процесс на передний план командой `fg`;
    
- продолжить в фоне командой `bg`.
    

### Ещё полезные сочетания

- **`Ctrl + A`**: перейти в начало строки.
    
- **`Ctrl + E`**: перейти в конец строки.
    
- **`Ctrl + U`**: удалить строку слева от курсора.
    
- **`Ctrl + K`**: удалить строку справа от курсора.
    
- **`Ctrl + L`**: очистить экран, аналог `clear`.
    
- **`Tab`**: автодополнение команды, пути или имени файла.

## 3. Синтаксический сахар и полезные команды

- **`!!`**: повтор последней команды. Часто используется как:
    

```bash
sudo !!
```

если забыл запустить команду с `sudo`.

- **`{}` (фигурные скобки)**: работа со списками и диапазонами. Например:
    

```bash
touch test{1..5}
```

создаст файлы `test1`, `test2`, `test3`, `test4`, `test5`.

Можно и так:

```bash
mkdir -p project/{src,bin,doc}
```

Это сразу создаст несколько директорий.

- **`watch`**: циклическое выполнение команды.  
    Пример:
    

```bash
watch -n 1 free -h
```

Команда будет показывать состояние памяти раз в секунду.

- **`jobs`**: просмотр списка фоновых и остановленных задач текущей оболочки.
    
- **`fg`** и **`bg`**:
    
    - `fg` — вернуть задачу на передний план;
        
    - `bg` — продолжить выполнение задачи в фоне.
        

### Дополнительно полезно знать

- **`history`** — показать историю команд.
    
- **`!номер`** — выполнить команду по её номеру из истории.
    
- **`!строка`** — выполнить последнюю команду, начинающуюся с указанной строки.
    

Примеры:

```bash
history
!105
!ssh
```

## 4. Потоки данных и перенаправление

В Bash существует три основных потока:

1. **stdin (0)** — поток ввода.
    
2. **stdout (1)** — стандартный поток вывода.
    
3. **stderr (2)** — поток ошибок.
    

Это стандартные файловые дескрипторы процесса.

### Символы перенаправления

- **`>`** — перенаправление вывода в файл с перезаписью:
    

```bash
echo hello > file.txt
```

- **`>>`** — добавление вывода в конец файла:
    

```bash
echo world >> file.txt
```

- **`2>`** — перенаправление только потока ошибок:
    

```bash
ls /no/such/path 2> errors.txt
```

- **`2>&1`** — объединение потока ошибок с основным потоком вывода:
    

```bash
command > all.txt 2>&1
```

- **`|` (pipe, конвейер)** — передача `stdout` одной команды на `stdin` другой:
    

```bash
ps aux | grep bash
```

### Дополнительные примеры

Подать файл на вход программе:

```bash
sort < names.txt
```

Скрыть ошибки:

```bash
ls /no/such/path 2>/dev/null
```

Отправить и вывод, и ошибки в `/dev/null`:

```bash
command >/dev/null 2>&1
```

Посчитать количество строк:

```bash
cat file.txt | wc -l
```

Хотя чаще пишут без лишнего `cat`:

```bash
wc -l < file.txt
```

## 5. Утилиты для работы с текстом

### Grep

`grep` используется для поиска строк по шаблону.

Полезные ключи:

- `-E` — расширенные регулярные выражения;
    
- `-i` — игнорировать регистр;
    
- `-v` — инвертировать поиск;
    
- `-n` — показывать номера строк;
    
- `-r` — рекурсивный поиск по директории.
    

Символы:

- `^` — начало строки;
    
- `$` — конец строки;
    
- `.` — любой символ;
    
- `*` — любое количество повторений предыдущего шаблона.
    

Примеры:

```bash
grep root /etc/passwd
grep -i error logfile.txt
grep -v '^#' config.conf
grep -E 'bash|zsh' /etc/passwd
grep -rn "TODO" .
```

### AWK

`awk` — мощный инструмент для обработки текста по столбцам.

Примеры:

```bash
awk '{print $2}' file.txt
```

Вывести второй столбец.

Для файла `/etc/passwd`, где разделитель `:`:

```bash
awk -F: '{print $1}' /etc/passwd
```

Вывести имена пользователей.

Вывести имя пользователя и его shell:

```bash
awk -F: '{print $1, $7}' /etc/passwd
```

### SED

`sed` — потоковый редактор для замены текста и простых преобразований.

Синтаксис:

```bash
sed -i 's/что_меняем/на_что_меняем/g' файл
```

Примеры:

```bash
sed 's/cat/dog/g' file.txt
sed -i 's/127.0.0.1/0.0.0.0/g' config.txt
```

Удалить пустые строки:

```bash
sed '/^$/d' file.txt
```

Показать только строки с 5 по 10:

```bash
sed -n '5,10p' file.txt
```

### Ещё полезные текстовые утилиты

- **`cut`** — вырезать столбцы;
    
- **`sort`** — сортировка;
    
- **`uniq`** — убрать повторы;
    
- **`wc`** — подсчёт строк, слов, байтов;
    
- **`head`** и **`tail`** — начало и конец файла.
    

Примеры:

```bash
cut -d: -f1 /etc/passwd
sort names.txt
uniq names.txt
wc -l file.txt
tail -f /var/log/syslog
```

## 6. Виртуальные терминалы: Screen и Tmux

Они нужны, чтобы процессы продолжали работать после закрытия SSH-сессии, и чтобы удобно делить одно окно терминала на несколько панелей.

### Screen

- Запуск именованной сессии:
    

```bash
screen -S mysession
```

- Отключиться от сессии:  
    `Ctrl + A`, затем `D`
    
- Вернуться:
    

```bash
screen -r
```

- Посмотреть список сессий:
    

```bash
screen -ls
```

### Tmux

Более современный мультиплексор терминала.

- Запуск новой сессии:
    

```bash
tmux
```

- Запуск именованной сессии:
    

```bash
tmux new -s work
```

- Основная управляющая клавиша:  
    `Ctrl + B`
    
- Разделить окно вертикально:  
    `Ctrl + B`, затем `%`
    
- Разделить окно горизонтально:  
    `Ctrl + B`, затем `"`
    
- Перемещение между панелями:  
    `Ctrl + B`, затем стрелки
    
- Отключиться от сессии:  
    `Ctrl + B`, затем `D`
    
- Вернуться:
    

```bash
tmux attach
```

- Список сессий:
    

```bash
tmux ls
```

### Когда это полезно

Например, ты подключился по SSH к серверу, запустил долгий процесс:

```bash
python train.py
```

Если просто закрыть терминал, процесс может завершиться. Если запустить его внутри `screen` или `tmux`, можно потом вернуться в сессию и продолжить работу.

## 7. Полезные команды навигации и работы с файлами

Этого не было явно выделено в исходном конспекте, но для темы “консоль Linux” это почти всегда нужно знать.

- **`pwd`** — показать текущую директорию.
    
- **`ls`** — список файлов.
    
- **`cd`** — переход между директориями.
    
- **`mkdir`** — создать директорию.
    
- **`touch`** — создать пустой файл.
    
- **`cp`** — копирование.
    
- **`mv`** — перемещение или переименование.
    
- **`rm`** — удаление.
    
- **`find`** — поиск файлов.
    
- **`locate`** — быстрый поиск по базе.
    

Примеры:

```bash
pwd
ls -lah
cd /etc
mkdir testdir
touch file.txt
cp file.txt copy.txt
mv copy.txt newname.txt
rm newname.txt
find . -name "*.py"
```
