**Монтирование** — это процесс создания связи между файлом блочного устройства (из `/dev/`) и конкретным каталогом в системе.

**Основные принципы:**
*   **Загрузка системы:** Монтирование необходимо для старта ОС — ядро должно подмонтировать корневую файловую систему (`/`).
*   **Права доступа:** По умолчанию процедура монтирования доступна только пользователю **root**.
*   **Точка монтирования:** Каталог, к которому привязывается диск. Традиционно для временных устройств используется каталог **`/mnt/`**.
*   **Каталог lost+found:** Появляется на подмонтированных дисках. В него система складывает фрагменты данных, найденные при проверке целостности файловой системы.

**Работа с командами:**
*   **Подключение:** `mount [что_монтируем] [куда_монтируем]` (например, `mount /dev/sda1 /mnt`).
*   **Отключение:** `umount` (unmount). Можно указывать либо имя каталога, либо имя устройства. После размонтирования данные остаются на диске, но каталог становится обычным пустым каталогом.

*Примечание:* Технически работать с диском без монтирования можно, но это требует ручной обработки структуры данных, что крайне сложно для обычного пользователя.
