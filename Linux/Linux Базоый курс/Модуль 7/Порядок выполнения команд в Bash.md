Для эффективной работы важно понимать, что командная оболочка (Bash) интерпретирует строку с перенаправлением **справа налево**.

**Алгоритм работы Bash:**
1.  Сначала Bash видит символ перенаправления и имя файла.
2.  Файл подготавливается: если его нет — создается, если есть и используется `>`, он **обнуляется** (становится размером 0 байт).
3.  Только после подготовки файла выполняется сама команда (левая часть строки).

**Критическая ошибка новичка (Ловушка с логами):**
Команда `tail -n 50 log > log` не обкалывает лог, а **полностью его стирает**. Поскольку Bash обнуляет файл `log` еще до того, как команда `tail` успеет его прочитать, `tail` получает на вход уже пустой файл и ничего не выводит.
