В Linux универсальным интерфейсом взаимодействия программ являются потоки текстовых данных. Каждая запущенная программа имеет три стандартных потока, называемых дескрипторами.

**Типы потоков:**
1.  **STDIN (номер 0):** Стандартный поток ввода. Через него программа получает аргументы. По умолчанию подключен к **клавиатуре**.
2.  **STDOUT (номер 1):** Стандартный поток вывода. Сюда попадает вся корректно обработанная информация. По умолчанию направлен на **терминал (монитор)**.
3.  **STDERR (номер 2):** Стандартный поток ошибок. Сюда выводятся сообщения о сбоях в процессе работы. По умолчанию также направлен на **терминал**.

Разделение вывода на корректные данные и ошибки позволяет гибко управлять информацией, направляя их в разные места.
