
Папка **«e»** (`e/classes.py`) посвящена **типизации классов и их атрибутов** в Python.  
Здесь разбирается, как аннотировать поля экземпляра и как отделять их от **переменных класса** с помощью `ClassVar`.

---

## 1. Обычные атрибуты экземпляра

Для атрибутов экземпляра используется обычная аннотация:

```python
class ExampleClass:
    a: int
````

Это означает:

> у каждого объекта `ExampleClass` должен быть атрибут `a` типа `int`

### Пример

```python
example = ExampleClass()
example.a = 123
```

Это корректно, потому что `a` объявлен как `int`.

---

## 2. Ошибка присваивания неправильного типа

В примере есть строка:

```python
example.a = '123'
```

Это **ошибка с точки зрения типизации**, потому что:

- `a` объявлен как `int`
    
- `'123'` имеет тип `str`
    

Статический анализатор (`mypy`, `pyright`, IDE) должен это подсветить.

### Затем:

```python
example.a = 123
```

Это уже корректное присваивание.

---

## 3. `ClassVar` — переменная класса

Если атрибут должен принадлежать **не экземпляру**, а самому классу, используется `ClassVar`.

### Пример

```python
from typing import ClassVar

class ExampleClass:
    a: int
    b: ClassVar[int]
```

Это означает:

- `a` — обычный атрибут экземпляра
    
- `b` — переменная класса, общая для всех объектов
    

---

## 4. Разница между атрибутом экземпляра и переменной класса

### Атрибут экземпляра

Каждый объект хранит своё значение:

```python
obj1.a = 10
obj2.a = 20
```

У `obj1` и `obj2` значения могут быть разными.

---

### Переменная класса

Значение одно на весь класс:

```python
ExampleClass.b = 789
```

Все экземпляры будут видеть это значение, если не произошло затенение на уровне объекта.

---

## 5. Что не так с `example.b = 456`

В примере есть строка:

```python
example.b = 456
```

С точки зрения статической типизации это **неправильно**, потому что `b` объявлен как `ClassVar[int]`.

То есть анализатор понимает это так:

> `b` нужно менять через класс, а не через конкретный экземпляр

### Правильный способ:

```python
ExampleClass.b = 789
```

---

## 6. Почему `ClassVar` нужен

`ClassVar` помогает явно показать намерение:

- этот атрибут общий для всех объектов
- он не должен храниться как обычное состояние экземпляра
- его не нужно изменять через `obj.attr`

Это делает код:

- понятнее
- чище
- безопаснее при статическом анализе

---

## 7. Важный момент: `ClassVar` работает только для анализаторов

`ClassVar` **не меняет поведение Python в рантайме**.

То есть Python сам по себе не запретит написать:

```python
example.b = 456
```

Но:

- `mypy`
- `pyright`
- IDE

будут считать это ошибкой проектирования.

Именно поэтому `ClassVar` — это **инструкция для типизатора**, а не механизм защиты во время выполнения.

---

## 8. Разбор примера построчно

Исходный код:

```python
from typing import ClassVar


class ExampleClass:
    a: int
    b: ClassVar[int]

example = ExampleClass()
example.a = '123'
example.a = 123

example.b = 456
ExampleClass.b = 789
```

---

### Что здесь происходит

#### Импорт

```python
from typing import ClassVar
```

Импортируется специальная конструкция для переменных класса.

---

#### Объявление класса

```python
class ExampleClass:
    a: int
    b: ClassVar[int]
```

- `a` — поле экземпляра типа `int`
- `b` — переменная класса типа `int`

---

#### Создание объекта

```python
example = ExampleClass()
```

Создаётся экземпляр класса.

---

#### Ошибка

```python
example.a = '123'
```

Неверно, потому что `a` должен быть `int`, а здесь `str`.

---

#### Корректно

```python
example.a = 123
```

Теперь тип совпадает.

---

#### Неверно для `ClassVar`

```python
example.b = 456
```

Статический анализатор должен считать это ошибкой, так как `b` — переменная класса.

---

#### Правильно

```python
ExampleClass.b = 789
```

Это корректное изменение значения `ClassVar`.

---

## 9. `Self` — полезное дополнение для методов классов

В материалах по этой теме также часто упоминается `Self`.

`Self` появился в Python 3.11 и используется, когда метод возвращает **экземпляр текущего класса**.

### Пример

```python
from typing import Self

class Builder:
    def set_value(self, x: int) -> Self:
        self.x = x
        return self
```

### Зачем это нужно

Это особенно удобно:

- для fluent interface
- для chain-вызовов
- для `classmethod`, возвращающих объект класса
- при наследовании

Преимущество `Self` в том, что он корректно работает и для подклассов.

---

## 10. Итог

Папка **«e»** учит различать два вида атрибутов:

### Атрибуты экземпляра

```python
a: int
```

Хранятся у конкретного объекта.

### Переменные класса

```python
b: ClassVar[int]
```

Принадлежат самому классу и не должны назначаться через экземпляр.

---

## 11. Главное, что нужно запомнить

- `a: int` — это поле экземпляра
- `b: ClassVar[int]` — это поле класса
- `example.a = '123'` — ошибка типов
- `example.a = 123` — корректно
- `example.b = 456` — ошибка с точки зрения `ClassVar`
- `ExampleClass.b = 789` — правильно

---

## Полный пример

```python
from typing import ClassVar


class ExampleClass:
    a: int
    b: ClassVar[int]


example = ExampleClass()
example.a = '123'
example.a = 123

example.b = 456
ExampleClass.b = 789
```
