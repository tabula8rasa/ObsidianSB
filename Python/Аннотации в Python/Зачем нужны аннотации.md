
Папка **«a»** в репозитории содержит файл `typed_vs_untyped.py`, который служит введением в систему аннотаций. Его основная цель — наглядное **сравнение кода без типов и с ними**, чтобы показать, как типизация улучшает читаемость и помогает находить ошибки до запуска программы.

Ниже представлен подробный конспект по материалам этой папки и пояснениям из видео.

### 1. Проблема нетипизированного кода

В примере без аннотаций (функция `process_user_data`) разработчик сталкивается с неопределенностью:

- **Неясные типы**: По названию `user_id` можно догадаться, что это число, но это может быть и строка (UUID). Непонятно, что такое `tags` — список строк или одна строка с разделителями.
- **Отсутствие подсказок**: IDE не может предложить методы объекта (например, методы строки `.upper()` или `.capitalize()`), так как не знает, что именно находится в переменной.
- **Скрытые ошибки**: Ошибка обнаружится только во время выполнения (runtime), если в функцию передадут некорректные данные.

### 2. Пример типизированного кода

Типизированная версия функции (в видео — `process`) наглядно демонстрирует современные стандарты Python:

```python
def process(
    user_id: int | str,            # Может быть числом или строкой
    name: str,                     # Строго строка
    is_active: bool,               # Логическое значение
    tags: list[str],               # Список, содержащий только строки
    metadata: dict[str, Any] | None = None  # Опциональный словарь (или None)
) -> dict[str, Any]:               # Возвращает словарь
    ...
```

**Ключевые элементы синтаксиса:**

- **Переменные**: Аннотируются через двоеточие (`name: str`).
- **Возвращаемое значение**: Указывается через стрелку после аргументов (`-> dict[...]`).
- **Оператор `|` (Union)**: Позволяет указать несколько допустимых типов (например, `int | str`). Этот синтаксис доступен начиная с Python 3.10.
- **Generic-коллекции**: `list[str]` указывает не просто на список, а на список конкретных элементов — строк.

### 3. Преимущества использования аннотаций

На примере кода из папки «a» выделяются три главных плюса:

1. **Документация кода**: Аннотации служат «внутренней подсказкой» для других разработчиков и для вас в будущем, семантически объясняя, какие данные ожидаются.
2. **Поддержка IDE**: Редактор кода (PyCharm, VS Code) подсвечивает ошибки сразу при написании. Если передать `int` там, где ожидается `str`, IDE выдаст предупреждение.
3. **Статический анализ**: С помощью инструмента **mypy** можно проверить весь проект на ошибки типов, не запуская сам код.

### 4. Важное замечание о Runtime

Согласно официальной документации Python, **интерпретатор не проверяет типы во время выполнения**. Аннотации — это метаданные для инструментов (IDE, линтеры, mypy), а сам Python продолжит работать с объектами так же, как и раньше.

Для автоматического добавления типов в уже существующий нетипизированный код в репозитории предлагается использовать инструмент **MonkeyType**, который анализирует типы данных прямо во время работы программы.
