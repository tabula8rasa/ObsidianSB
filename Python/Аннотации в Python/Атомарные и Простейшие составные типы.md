
Папка **«b»** (`b/simple_types.py`) посвящена **базовым типам данных** и **аннотациям типов** в Python. Она показывает, как описывать типы переменных, коллекций, функций и констант с помощью встроенного синтаксиса языка и модуля `typing`.

Ниже — объединённый и структурированный конспект по этим материалам.

---
## 1. Простые типы переменных

Для аннотирования обычных переменных используются встроенные типы Python:

- `str` — строки
- `int` — целые числа
- `float` — числа с плавающей точкой
- `bool` — логические значения `True` / `False`

### Пример
```python
name: str = "Python"
age: int = 30
price: float = 99.99
is_available: bool = True
````

Даже если IDE часто умеет выводить тип автоматически, явная аннотация помогает:

- сделать код понятнее
- раньше находить ошибки
- улучшить автодополнение и проверки в IDE

---

## 2. Коллекции и составные типы

Коллекции в Python можно аннотировать, указывая тип элементов внутри.

### 2.1. Списки (`list`)

```python
users: list = ["Alice", "Bob"]         # слишком общий вариант
users: list[str] = ["Alice", "Bob"]    # Python 3.9+
users: List[str] = ["Alice", "Bob"]    # Python 3.6–3.8
```

> Начиная с Python 3.9, предпочтительнее использовать встроенный `list[str]`, а не `typing.List[str]`.

---

### 2.2. Кортежи (`tuple`)

#### Кортеж фиксированной длины

```python
users: tuple[str, int] = ("Alice", 1)
```

Это значит:

- первый элемент — `str`
    
- второй элемент — `int`
    

#### Кортеж произвольной длины

```python
users: tuple[str, ...] = ("Alice", "a", "2", "3")
```

Запись `tuple[str, ...]` означает:  
**кортеж любой длины, состоящий только из строк**.

---

### 2.3. Словари (`dict`)

```python
counts: dict[str, int] = {"apples": 10, "oranges": 5}  # Python 3.9+
counts: Dict[str, int] = {"apples": 10, "oranges": 5}  # Python 3.6–3.8
```

У словаря указываются:

- тип ключа
    
- тип значения
    

То есть `dict[str, int]` означает:  
**ключи — строки, значения — целые числа**.

---

## 3. `Optional` и `Union`

Иногда переменная может содержать не один тип, а несколько. Для этого используются объединения типов.

---

### 3.1. `Optional`

```python
user_id: Optional[int] = None
user_id: int | None = None
```

Обе записи означают одно и то же:

> переменная может быть либо `int`, либо `None`

Фактически:

```python
Optional[int] == Union[int, None]
```

---

### 3.2. `Union`

```python
user_id: Union[int, str] = 1
user_id: int | str = "abc123"
```

Это означает:

> переменная может быть либо `int`, либо `str`

#### Старый и новый синтаксис

- старый: `Union[int, str]`
    
- современный (Python 3.10+): `int | str`
    

Современный вариант короче и читается легче.

---

## 4. Типизация функций

При аннотировании функций указываются:

- типы аргументов
    
- тип возвращаемого значения
    

### Пример

```python
def greet(name: str) -> str:
    return f"Hello, {name}!"
```

Здесь:

- `name: str` — аргумент должен быть строкой
- `-> str` — функция возвращает строку
---

### Пример с несколькими аргументами

```python
def add_user(name: str, age: int, is_active: bool = True) -> None:
    """Добавляет пользователя в систему."""
    print(f"Added user {name}, age {age}, active: {is_active}")
```

Здесь:

- `name` — строка
- `age` — целое число
- `is_active` — логическое значение
- `-> None` — функция ничего не возвращает

---

### Ещё пример

```python
def multiply(a: int, b: int) -> int:
    return a * b
```

---

## 5. `Any` — любой тип

Если тип заранее неизвестен или может быть абсолютно любым, используют `Any`.

```python
def get_data():
    ...

data: Any = get_data()
```

`Any` полезен, когда:

- тип действительно невозможно точно описать
- идёт работа с внешними библиотеками
- нужен временный компромисc

Но важно помнить:

> `Any` фактически отключает строгую проверку типов

Поэтому злоупотреблять им не рекомендуется.

---

## 6. `Final` — константы

Для переменных, значение которых **не должно изменяться после присвоения**, используется `Final`.

```python
from typing import Final

final_list: Final[object] = object()
```

Смысл такой:

- статический анализатор (`mypy`, `pyright`, IDE) будет считать переменную константой
- повторное присваивание пометит как ошибку

### Пример

```python
final_list: Final[object] = object()
print(final_list.__doc__)

final_list = [1, 2, 3]  # Ошибка для анализатора типов
```

Важно:

> Сам Python в рантайме это обычно не запрещает. Ошибку показывает именно статический анализатор.

---

## 7. Импорты из `typing`

Часто используются такие типы и конструкции:

```python
from typing import List, Dict, Optional, Union, Any, Final
```

### Для чего они нужны

- `List` — список
- `Dict` — словарь
- `Optional` — тип или `None`
- `Union` — объединение нескольких типов
- `Any` — любой тип
- `Final` — константа

---

## 8. Важное различие: встроенные типы и типы из `typing`

### Современный стиль (Python 3.9+)

```python
list[str]
dict[str, int]
tuple[str, int]
```

### Совместимый стиль для старых версий

```python
List[str]
Dict[str, int]
Tuple[str, int]
```

Если код пишется под Python 3.9+, обычно лучше использовать встроенный синтаксис.

---

## 9. Практический смысл аннотаций типов

Аннотации нужны не для самого Python, а в первую очередь для:

- IDE и автодополнения
- статического анализа (`mypy`, `pyright`)
- улучшения читаемости кода
- раннего поиска ошибок до запуска программы

Например, если переменная объявлена как:

```python
name: str = "Python"
```

то попытка потом написать:

```python
name = 123
```

будет помечена анализатором как подозрительная.

## Полный пример из материалов

```python
from typing import List, Dict, Optional, Union, Any, Final

# Простые типы
name: str = "Python"
age: int = 30
price: float = 99.99
is_available: bool = True

# Составные типы
users: list = ["Alice", "Bob"]  # Но лучше указать тип элементов!
users: list[str] = ["Alice", "Bob"]  # Python 3.9+
users: List[str] = ["Alice", "Bob"]  # Python 3.6-3.8 (с импортом)

users: tuple[str, int] = ("Alice", 1)  # два элемента: строка и число
users: tuple[str, ...] = ("Alice", "a", "2", "3")  # много строк

# Словари и другие коллекции
counts: dict[str, int] = {"apples": 10, "oranges": 5}  # Python 3.9+
counts: Dict[str, int] = {"apples": 10, "oranges": 5}  # Python 3.6-3.8

# None и опциональные значения
user_id: Optional[int] = None  # int или None
user_id: int | None = None     # int или None

# Union — объединение типов
user_id: Union[int, str] = 1   # int или str
user_id: int | str = "abc123"  # То же самое, но короче

def greet(name: str) -> str:
    return f"Hello, {name}!"

def add_user(name: str, age: int, is_active: bool = True) -> None:
    """Добавляет пользователя в систему."""
    print(f"Added user {name}, age {age}, active: {is_active}")

def get_data():
    ...

# Any — любой тип
data: Any = get_data()

# Final — константа
final_list: Final[object] = object()
print(final_list.__doc__)

final_list = [1, 2, 3]  # Ошибка для статического анализатора
```
