
Папка **«d»** (`d/callable.py`) посвящена типизации **вызываемых объектов** и **асинхронных интерфейсов** в Python.  
Основные инструменты здесь — `Callable` и `Awaitable`.

Они нужны, когда:

- функция передаётся как аргумент в другую функцию
- нужно описать callback
- нужно аннотировать корутину или объект, который можно `await`
- важно, чтобы статический анализатор заранее проверял совместимость сигнатур

---

## 1. `Callable` — типизация функций как объектов

В Python функции являются объектами первого класса. Их можно:

- сохранять в переменные
- передавать в другие функции
- возвращать из функций

Чтобы описать тип такой функции, используется `Callable`.

### Синтаксис

```python
Callable[[ТипАргумента1, ТипАргумента2], ТипВозвращаемогоЗначения]
````

### Пример

```python
Callable[[int, int], int]
```

Это означает:

> функция принимает два `int` и возвращает `int`

---

## 2. Пример с `apply_twice`

```python
from typing import Callable

def apply_twice(
    func: Callable[[int, int], int],
    x: int
) -> int:
    func(x, x)
    return func(x, x)
```

### Что здесь указано

- `func` — это функция, принимающая два `int`
- возвращает она тоже `int`
- `x` — обычное целое число
- сама `apply_twice` возвращает `int`

---

## 3. Проверка совместимости функций

### Подходящая функция

```python
def example_function(a: int, b: int) -> int:
    ...
```

Она подходит, потому что сигнатура совпадает:

- `a: int`
- `b: int`
- возвращает `int`

Можно передать так:

```python
apply_twice(example_function, 2)
```

---

### Неподходящая функция

```python
def example_function2(a: str, b: str) -> str:
    ...
```

Она **не подходит**, потому что ожидается:

```python
Callable[[int, int], int]
```

а передаётся:

```python
Callable[[str, str], str]
```

То есть:

- аргументы не те (`str` вместо `int`)
- возвращаемый тип не тот (`str` вместо `int`)

### Ошибочный вызов

```python
apply_twice(example_function2, 3)
```

Статический анализатор (`mypy`, `pyright`) должен подсветить это как ошибку типов.

---

## 4. `Callable[..., T]` — функция с произвольными аргументами

Если точная сигнатура не важна или функция может принимать любые аргументы, можно использовать многоточие:

```python
Callable[..., int]
```

Это означает:

> вызываемый объект с любыми аргументами, но возвращающий `int`

### Но важно

Такой вариант менее строгий.  
Он полезен, когда:

- аргументы заранее неизвестны
- нужна общая аннотация
- точность сигнатуры не критична

---

## 5. `Callable` — это не только обычные функции

Под `Callable` подходят не только функции, объявленные через `def`.

### Также подходят:

- лямбда-функции
- классы
- экземпляры классов с методом `__call__`

---

### Пример вызываемого объекта

```python
class Adder:
    def __call__(self, a: int, b: int) -> int:
        return a + b
```

Теперь объект этого класса тоже можно передавать туда, где ожидается `Callable[[int, int], int]`:

```python
adder = Adder()
apply_twice(adder, 5)
```

Потому что `adder(...)` работает как функция.

---

## 6. `Awaitable` — типизация асинхронных результатов

Для асинхронного кода используется `Awaitable`.

Он описывает объект, который можно использовать с `await`.

### Смысл

```python
Awaitable[int]
```

означает:

> объект, который после `await` даст `int`

---

## 7. Пример асинхронных функций

```python
async def async_example(a: int, b: int) -> int:
    ...

async def async_example_string(a: int, b: int) -> str:
    ...
```

Здесь:

- `async_example(...)` возвращает awaitable-объект с результатом `int`
- `async_example_string(...)` возвращает awaitable-объект с результатом `str`

---

## 8. Функция, принимающая `Awaitable[int]`

```python
async def call(coro: Awaitable[int]):
    await coro
```

Это означает:

- `coro` должен быть awaitable-объектом
- после `await` он должен вернуть `int`

---

### Корректный вызов

```python
call(async_example(1, 2))
```

Потому что:

- `async_example(1, 2)` даёт `Awaitable[int]`

---

### Ошибочный вызов

```python
call(async_example_string(1, 2))
```

Потому что:

- `async_example_string(1, 2)` даёт `Awaitable[str]`
- а ожидается `Awaitable[int]`

Статический анализатор должен сообщить об ошибке несовпадения типов.

---

## 9. Почему это полезно

Такая типизация помогает ловить ошибки **до запуска программы**.

Например:

- передали не ту callback-функцию
- спутали синхронную и асинхронную функцию
- ожидали `int`, а получили `str`
- неверно описали интерфейс вычисления

Это особенно важно:

- в библиотеках
- в декораторах
- в асинхронных системах
- в коде с callback-архитектурой

---

## 10. Ограничения стандартного `Callable`

Обычный `Callable` хорош, но у него есть ограничения.

Он плохо описывает:

- `keyword-only` аргументы
- сложные сигнатуры
- перегруженные функции
- функции с `*args` и `**kwargs`, если нужно точно сохранить типы
- нестандартные вызываемые интерфейсы

---

## 11. Когда лучше использовать `Protocol`

Если сигнатура сложная, вместо `Callable` лучше использовать `Protocol` с методом `__call__`.

### Пример

```python
from typing import Protocol

class IntOperation(Protocol):
    def __call__(self, a: int, b: int) -> int:
        ...
```

Теперь можно описывать более сложные вызываемые объекты через структурную типизацию.

Это даёт больше гибкости, чем обычный `Callable`.

---

## 12. Итог

Папка **«d»** показывает, как типизировать:

- функции как значения
- callback-функции
- вызываемые объекты
- асинхронные результаты

### Основные инструменты:

- `Callable[[...], T]` — функция с заданной сигнатурой
- `Callable[..., T]` — функция с любыми аргументами
- `Awaitable[T]` — объект, который после `await` возвращает `T`

Это важная часть современной типизации Python, особенно при работе с высокоуровневыми интерфейсами и асинхронным кодом.

## Полный пример из материалов

```python
from typing import Callable, Awaitable


def apply_twice(
    func: Callable[
        [int, int],  # аргументы функции
        int          # возвращаемое значение
    ],
    x: int
) -> int:
    func(x, x)
    return func(x, x)


def example_function(a: int, b: int) -> int:
    ...


def example_function2(a: str, b: str) -> str:
    ...


apply_twice(example_function, 2)
apply_twice(example_function2, 3)


async def async_example(a: int, b: int) -> int:
    ...


async def async_example_string(a: int, b: int) -> str:
    ...


async def call(coro: Awaitable[int]):
    await coro


call(async_example(1, 2))
call(async_example_string(1, 2))
```
