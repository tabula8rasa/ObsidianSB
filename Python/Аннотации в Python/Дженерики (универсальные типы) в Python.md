
Папка **«f»** (`f/generic.py`) посвящена **дженерикам** — универсальным типам в Python.  
Это один из самых полезных инструментов типизации: он позволяет писать код, который работает с разными типами данных, но при этом сохраняет строгую связь между входными и выходными типами.

---

## 1. Что такое дженерики

Дженерики нужны, когда:

- функция или класс могут работать с разными типами
- но хочется, чтобы анализатор типов **сохранял информацию** о конкретном типе

То есть идея такая:

> мы не знаем тип заранее, но хотим, чтобы он был согласован во всех местах использования

---

## 2. Пример функции с параметром типа

```python
from typing import Sequence

def first[T](collection: Sequence[T]) -> T:
    """Возвращает первый элемент последовательности."""
    return collection[0]
````

### Что здесь происходит

- `[T]` — объявление параметра типа
- `collection: Sequence[T]` — на вход подаётся последовательность элементов типа `T`
- `-> T` — возвращается элемент того же типа `T`

---

### Использование

```python
num = first([1, 2, 3])          # mypy выведет int
name = first(["Alice", "Bob"])  # mypy выведет str
```

### Почему это полезно

Если на вход:

- `list[int]` → результат `int`
- `list[str]` → результат `str`

То есть функция универсальная, но тип результата не теряется.

---

## 3. `Sequence[T]` вместо `list[T]`

В примере используется:

```python
Sequence[T]
```

а не просто `list[T]`.

Это удобно, потому что `Sequence` — более общий интерфейс.  
Под него подходят:

- `list`
- `tuple`
- `str`
- другие последовательности

То есть функция становится гибче.

---

## 4. Ограничения типов (constraints)

Иногда дженерик должен работать не с любым типом, а только с несколькими разрешёнными.

### Пример

```python
def concat[T: (str, bytes)](a: T, b: T) -> T:
    return a + b
```

---

### Что означает `[T: (str, bytes)]`

Это ограничение говорит:

> `T` может быть только `str` или `bytes`

Причём важно не просто “один из них”, а то, что **оба аргумента должны быть одного и того же типа `T`**.

---

### Корректные вызовы

```python
result1 = concat("Hello, ", "world!")      # OK
result2 = concat(b"Hello, ", b"world!")    # OK
```

---

### Ошибочный вызов

```python
result3 = concat("Hello, ", b"world!")  # Ошибка
```

Почему ошибка:

- первый аргумент — `str`
- второй аргумент — `bytes`

А функция требует, чтобы оба были **одного и того же типа `T`**.

Чтобы иметь возможность передать два аргумента каждый из которых или `str` или `bytes` по отдельности, нужно ввести новый параметр функции

```python
def concat[T: (str, bytes), F: (str, bytes)](a: T, b: F) -> T:
    return ...
```

Тогда при таком вызове ошибки не будет
```python
result3 = concat("Hello, ", b"world!") 
```


## 5. Дженерик-классы

Дженерики можно использовать не только в функциях, но и в классах.

### Пример стека

```python
class Stack[T]:
    def __init__(self) -> None:
        self.items: list[T] = []

    def push(self, item: T) -> None:
        self.items.append(item)

    def pop(self) -> T | None:
        return self.items.pop() if self.items else None

    def peek(self) -> T | None:
        return self.items[-1] if self.items else None
```

---

## 6. Как работает `Stack[T]`

Здесь `T` — параметр типа всего класса.

Это значит:

- `Stack[int]` — стек целых чисел
- `Stack[str]` — стек строк
- `Stack[User]` — стек объектов `User`

---

### Методы

#### `push`

```python
def push(self, item: T) -> None:
```

Принимает только элементы типа `T`.

#### `pop`

```python
def pop(self) -> T | None:
```

Возвращает:

- элемент типа `T`, если стек не пуст
- `None`, если стек пуст

#### `peek`

```python
def peek(self) -> T | None:
```

Смотрит на верхний элемент без удаления.

---

## 7. Использование дженерик-класса

### Стек целых чисел

```python
int_stack = Stack[int]()
int_stack.push(1)
int_stack.push(2)
int_stack.push("4")  # Ошибка! Ожидается int
```

Поскольку это `Stack[int]`, метод `push` должен принимать только `int`.

---

### Стек строк

```python
str_stack = Stack[str]()
str_stack.push("hello")
str_value = str_stack.pop()  # тип: str | None
```

Здесь `mypy` поймёт, что `str_value` имеет тип:

```python
str | None
```

---

## 8. Новый синтаксис дженериков (Python 3.12+)

Начиная с Python 3.12, появился новый синтаксис:

### Для функций

```python
def first[T](collection: Sequence[T]) -> T:
    ...
```

### Для классов

```python
class Stack[T]:
    ...
```

Это современный и более короткий способ записи.

---

## 9. Старый синтаксис (до Python 3.12)

До Python 3.12 использовались `TypeVar` и `Generic`.

### Пример

```python
from typing import TypeVar, Generic, Sequence

T = TypeVar("T")

def first(collection: Sequence[T]) -> T:
    return collection[0]


class Stack(Generic[T]):
    def __init__(self) -> None:
        self.items: list[T] = []
```

### Разница

Новый синтаксис:

- короче
- чище
- легче читается

Но старый всё ещё нужен:

- для старых версий Python
- в проектах, где ещё не перешли на 3.12+    

---

## 10. Constraints и Bound — не одно и то же

В материалах важно различать два похожих механизма.

### Constraints

```python
[T: (str, bytes)]
```

Означает:

> `T` может быть только одним из перечисленных типов

---

### Bound

Идея bound — задать верхнюю границу:

```python
[T: float]
```

Это означает:

> `T` должен быть `float` или его подтипом

То есть:

- constraints = конкретный список допустимых типов
- bound = верхняя граница иерархии типов

---

## 11. Зачем вообще нужны дженерики

Без дженериков пришлось бы писать отдельные версии:

- `first_int`
- `first_str`
- `StackInt`
- `StackStr`

С дженериками мы пишем один универсальный код, а тип подставляется автоматически.

Это даёт:

- меньше дублирования
- более точные подсказки IDE
- меньше ошибок
- лучшее проектирование API    

---

## 12. Главное преимущество

Дженерики позволяют выразить мысль:

> функция или класс работают с “каким-то типом”, и этот тип должен сохраняться согласованным

Например:

- если дали `Sequence[int]`, вернётся `int`
- если создали `Stack[str]`, туда нельзя положить `int`    

---

## 13. Дополнительные возможности

В более сложных случаях можно использовать несколько параметров типа:

```python
def process[T, R](data: T) -> R:
    ...
```

Также существуют специальные инструменты для более сложной типизации:

- `TypeVarTuple`
- `ParamSpec`

Они нужны для:

- `*args`
- `**kwargs`
- декораторов
- сложных сигнатур функци    

Обычно это уже следующий уровень типизации.

---

## 14. Итог

Папка **«f»** показывает, как писать **универсальный, но строго типизированный код**.

Основные идеи:

- дженерики позволяют работать с разными типами без потери точности
- параметр типа связывает вход и выход
- можно ограничивать допустимые типы
- можно создавать дженерик-функции и дженерик-классы
- новый синтаксис Python 3.12 делает это гораздо удобнее

---

## Полный пример из материалов

```python
# List[T] — это дженерик-класс, где T — параметр типа
from typing import Sequence

def first[T](collection: Sequence[T]) -> T:
    """Возвращает первый элемент последовательности."""
    return collection[0]

# Использование
num = first([1, 2, 3])  # mypy выведет тип int
name = first(["Alice", "Bob"])  # mypy выведет тип str


def concat[T: (str, bytes)](a: T, b: T) -> T:
    return a + b

result1 = concat("Hello, ", "world!")  # OK, оба аргумента str
result2 = concat(b"Hello, ", b"world!")  # OK, оба аргумента bytes
result3 = concat("Hello, ", b"world!")  # Ошибка! Разные типы


# Новый синтаксис (Python 3.12+)
class Stack[T]:
    def __init__(self) -> None:
        self.items: list[T] = []

    def push(self, item: T) -> None:
        self.items.append(item)

    def pop(self) -> T | None:
        return self.items.pop() if self.items else None

    def peek(self) -> T | None:
        return self.items[-1] if self.items else None

# Использование
int_stack = Stack[int]()
int_stack.push(1)
int_stack.push(2)
int_stack.push("4")  # Ошибка! Ожидается int

str_stack = Stack[str]()
str_stack.push("hello")
str_value = str_stack.pop()  # mypy выведет тип str | None
```
