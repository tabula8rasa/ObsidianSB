Инвертированный индекс — это структура данных, которая хранит не **документ → слова**, а наоборот: **слово → список документов, где оно встречается**.

Простой пример.

Есть 3 документа:

- Doc1: `я люблю базы данных`
    
- Doc2: `базы данных это важно`
    
- Doc3: `я люблю python`
    

Обычный взгляд на данные:

- Doc1 → я, люблю, базы, данных
    
- Doc2 → базы, данных, это, важно
    
- Doc3 → я, люблю, python
    

Инвертированный индекс:

- `я` → Doc1, Doc3
    
- `люблю` → Doc1, Doc3
    
- `базы` → Doc1, Doc2
    
- `данных` → Doc1, Doc2
    
- `python` → Doc3
    

Почему он нужен:  
когда ты ищешь слово, системе не надо читать все документы целиком. Она просто смотрит в индекс и сразу получает список подходящих документов.

Как это работает при поиске:  
если запрос `базы AND данных`, то берутся:

- `базы` → Doc1, Doc2
    
- `данных` → Doc1, Doc2
    

Пересечение списков = Doc1, Doc2.

Если запрос `python`, то сразу получаем Doc3.

Обычно в инвертированном индексе хранят не только список документов, но и дополнительные данные:

- сколько раз слово встретилось
    
- в каких позициях оно стоит
    
- в каком поле оно найдено, например в заголовке или тексте
    

Например:

- `python` → `(Doc3, позиции: [3], count: 1)`
    

Это нужно для:

- быстрого полнотекстового поиска
    
- поиска фраз, например `"базы данных"`
    
- ранжирования результатов по релевантности
    

Где используется:

- поисковики
    
- Elasticsearch, OpenSearch, Lucene
    
- полнотекстовый поиск в БД
    
- поиск по логам, статьям, сообщениям

```python
import re
from collections import defaultdict


class InvertedIndex:
    def __init__(self):
        # слово -> set(id документов)
        self.index = defaultdict(set)
        # id документа -> исходный текст
        self.documents = {}

    def _tokenize(self, text: str) -> list[str]:
        # очень простая токенизация:
        # приводим к нижнему регистру и вытаскиваем слова
        return re.findall(r"\w+", text.lower())

    def add_document(self, doc_id: int, text: str) -> None:
        self.documents[doc_id] = text
        words = self._tokenize(text)

        for word in words:
            self.index[word].add(doc_id)

    def search(self, word: str) -> list[int]:
        word = word.lower()
        return sorted(self.index.get(word, set()))

    def search_and(self, *words: str) -> list[int]:
        if not words:
            return []

        result = self.index.get(words[0].lower(), set()).copy()

        for word in words[1:]:
            result &= self.index.get(word.lower(), set())

        return sorted(result)

    def search_or(self, *words: str) -> list[int]:
        result = set()

        for word in words:
            result |= self.index.get(word.lower(), set())

        return sorted(result)


# пример использования
idx = InvertedIndex()

idx.add_document(1, "Я люблю базы данных")
idx.add_document(2, "Базы данных это важно")
idx.add_document(3, "Я люблю Python")

print("Поиск 'люблю':", idx.search("люблю"))
print("Поиск AND('базы', 'данных'):", idx.search_and("базы", "данных"))
print("Поиск OR('python', 'важно'):", idx.search_or("python", "важно"))

print("\nНайденные документы для слова 'люблю':")
for doc_id in idx.search("люблю"):
    print(doc_id, "->", idx.documents[doc_id])
```
